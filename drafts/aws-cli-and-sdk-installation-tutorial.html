Introduction¶

AWS provides multiple ways to control its resources. One of the fastest and the most effective way is the AWS CLIs and SDKs. The best benefit of using CLI or SDKs it that you can automate complex work flow.

Depending you expertise, you can use bash script to control the AWS resources. You can use multiple programming languages, like .NET, Python, Go, etc to control the AWS resources. If you are not aware of both bash script or programming language you can still you Postman client to make HTTP API Calls.

This article will make you comfortable using the command line, programming or the Postman client to control AWS resource. This simple knowledge will save you a large amount of time in the future as you can automate the mundane activity.

Come and explore the work of AWS CLIs, SDKs and Postman Client.

Prerequisite¶

If you are a aware of Python, then installation of CLI and SDK become very easy using the pip module and venv what is virtual environment. This also is cross platform. If you use Python, then the process becomes easier for Windows, Linux and MacOS.

There is a new version of AWS CLI V 2.x which do not support installation via pip. You will install only AWS CLI V 2.x via the package installer method mentioned below by AWS.

Install Python on Windows, Linux or macOS¶

There are numerous resource out there to install Python on these platform. I cannot do any improvement over these installation instruction.

Please follow this RealPython to install Python on your respective operating system.

Once you have followed the instruction mentioned above, please execute the below command to check the installation.

python --version

If you get any version information in return then you have successfully installed Python.

You should now proceed to install the next important package called pip.

Install PiP on Window, Linux or macOS¶

Once you have Python installed, the next step would be to install the package manager called pip. For Python 2 version >= 2.7.9 and Python 3 version >= 3.4 pip should be installed by default. You can check the installation by executing this command.

python -m pip --version

If you get any response like the version information understand that pip is already present. If not present you can use this documentation to install pip.

Installing PiP for Windows, Linux and macOS

Once you have pip installed, the next step is to create virtual environment using the package venv.

Create Virtual Environment¶

You can have separate environment for your Python installation. This helps in not polluting the global package installation.

Create a directory name aws-cli in you local machine and follow along. Change to this aws-cli directory.

Create a virtual environment

python -m venv .env

If no error comes then there will be a .env inside aws-cli. If we get an error of package not found, please install virtual environment package, using pip.

The .env is generally used to identify an environment folder, but you are free to use any name. 

python -m pip install --user virtualenv

Once you have your .env folder created now we can proceed with the AWS CLI and SDK installation.

You are all set to install AWS CLI and SDK.

AWS CLI¶

AWS CLI is a tool to manage multiple services. If you choose the Amazon Linux AMI, the AWS CLI is pre-installed.

AWS CLI is an open source tool. It helps to provide the same functionality which we achieve via the AWS Management Console. All AWS IaaS (Infrastructure as a Service) administration feature available via AWS Management Console should be available on AWS CLI in 180 days.

You get access to only the public APIs. While using AWS CLI, you do not have any secrete APIs to use, all the APIs are available in public.

AWS CLI comes in two version, the most recent version is 2.x, it can be used in production environment. The biggest difference between version 1.x and 2.x is that 2.x is not available as a Python package for installation through pip.

You have to use a separate installer to install the version 2.x. The support for AWS ClI v1 is still available.

There is an unofficial port of AWS ClI v2 on pypi called awscliv2.

If you want to install v2 directly jump to Install AWS CLI v2.

Install AWS CLI v1¶

You can install the AWS CLI v1 simply by activating the virtual environment inside aws-cli folder. The below command works on Linux/macOS. It may be little different on Windows.

source .env/bin/activate

Once the virtual environment is activated, you can execute this below command.

python -m pip install awscli

Once you have installed, it will show this at the end.

Successfully installed PyYAML-5.4.1 awscli-1.19.62 botocore-1.20.62 colorama-0.4.3 docutils-0.15.2 jmespath-0.10.0 pyasn1-0.4.8 python-dateutil-2.8.1 rsa-4.7.2 s3transfer-0.4.2 six-1.15.0 urllib3-1.26.4

If you see, while installing awscli it installs the above dependencies, one of that dependency is botocore. botocore is the base for even boto3.

You can verify the installation by running the command.

aws --version

If the version is successfully installed you will get an output as aws-cli/1.19.62 Python/3.7.5 Darwin/18.6.0 botocore/1.20.62.

You have awscli v1 installed.

Install AWS CLI v2¶

For macOS:-

Download the latest AWS CLI v2 packages from AWS CLI pkg Download.

Once downloaded, right click on the package file and open it using installer application, select the default options. At the end aws would be installed.

If you have both AWS CLI v1 and v2 installed then it will pick the executable which is first in the PATH variable. Confirm the installation by opening a new terminal and executing.

aws --version

You should get an output as 

aws-cli/2.2.1 Python/3.8.8 Darwin/18.6.0 exe/x86_64 prompt/off

The above output proves that we have the AWS CLI v2 installed. Now you should execute a sample command to see aws cli in action.

AWS SDK | Python | Boto3¶

SDKs take the complexity out of coding by providing language specific APIs for AWS service. AWS SDK is available in these various languages.

  • JavaScript
  • Python
  • PHP
  • .NET
  • Ruby
  • Java
  • Go
  • Node.js
  • C++

You can use any of the SDK for controlling the AWS resources. You will learn to use the Python SDK called Boto3..

The Python Boto3 provides two key Python packages.

  1. Botocore - Library providing low-level functionality shared with Python SDK and the AWS CLI
  2. Boto3 - The Python SDK.

Installation of Boto3 is very easy, activate the Python virtual environment, which we already did Install AWS CLI v1

Once you have activated the virtual environment, just execute this command.

python -m pip install boto3

Once the installation is done, it will give this message.

Successfully installed boto3-1.17.62

You can again execute the configure option, if not done as explained in AWS CLI configure AWS environment.

describe_images API using Python Boto3¶

Once installation is complete we can execute the same describe_images to get the details of the AMI.

import boto3
import pprint

ec2 = boto3.client("ec2")

ImageIds = ["ami-0d758c1134823146a"]

response = ec2.describe_images(ImageIds=ImageIds)
pprint.pprint(response[""])
